# 贡献指南

## 贡献方式

### 1. 概念贡献
- 在 `CONCEPTS/` 目录下创建新概念文件
- 文件名格式：`UMA-{3位编号}-{概念名}.md`
- 遵循概念模板：定义、原理、应用、示例、相关概念
- 提交Pull Request，由至少2名维护者评审

### 2. 实践指南
- 在 `PRACTICES/` 下分享具体创作方法
- 包括：技术栈、工作流、问题解决、最佳实践
- 鼓励附代码示例和可视化图表

### 3. 项目提交
- 在 `PROJECTS/` 下创建子目录
- 包含：README.md（项目描述）、代码、文档、演示
- 确保项目完整可运行（或提供在线演示链接）

### 4. 工具推荐
- 在 `TOOLS/` 下添加工具文档
- 包括：工具介绍、UMA应用场景、配置示例、优缺点分析

### 5. 社区讨论
- 使用GitHub Discussions发起话题
- 重要讨论存档到 `COMMUNITY/discussions/`
- 工作坊资料整理到 `COMMUNITY/workshops/`

## 工作流程
1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/amazing-concept`)
3. 提交更改 (`git commit -m 'Add some amazing concept'`)
4. 推送到分支 (`git push origin feature/amazing-concept`)
5. 开启Pull Request

## 编码规范
- Markdown文件遵循标准格式
- 代码示例注明语言和运行环境
- 所有外部链接确保可访问
- 图片使用相对路径，尺寸适中

## 行为准则
所有贡献者需遵守 [CODE_OF_CONDUCT.md](./CODE_OF_CONDUCT.md)
